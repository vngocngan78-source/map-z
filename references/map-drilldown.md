# 地图下钻规范

## 概述

map-z 支持四级地图下钻：全国 → 省 → 市 → 区县

## 层级定义

| 层级 | 名称 | adcode 格式 | 示例 |
|------|------|------------|------|
| 0 | 全国 | 100000 | 显示31个省份 |
| 1 | 省级 | XX0000 | 110000 = 北京 |
| 2 | 市级 | XXXX00 | 110100 = 北京市区 |
| 3 | 区县 | XXXXXX | 110101 = 东城区 |

## 数据来源

所有边界数据来自阿里云 DataV GeoJSON：

```
https://geo.datav.aliyun.com/areas_v3/bound/{adcode}_full.json
```

## 下钻交互规范

### 触发条件
- 用户点击地图上任意有数据的区域
- 该区域 adcode 末位为 0（还有下级）

### 下钻动画
- 显示加载 spinner
- 地图重新渲染新层级
- 视角自动缩放到下钻区域
- 面包屑更新

### 返回方式
- 点击顶部面包屑任意层级
- 点击"← 返回上级"按钮

### 飞线处理
- 下钻时隐藏飞线（仅全国层级显示）
- 返回全国时恢复显示

## 数据匹配规则

GeoJSON 中的地区名称需要标准化处理：

```javascript
const shortName = rawName.replace(
  /省|市|壮族自治区|回族自治区|维吾尔自治区|自治区|特别行政区/g, ''
)
```

然后与用户数据中的 name 字段做模糊匹配。
